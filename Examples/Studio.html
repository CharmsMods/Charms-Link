Digital Grain Studio

A GPU-accelerated image processing pipeline running directly in the browser. It utilizes WebGL2 frame buffers to perform real-time compositing, procedural noise generation, and algorithmic corruption without server-side processing.

Inspect Capabilities

01. Noise & Texture

Procedural generation based on fragment shaders. Not an overlay, but a calculated texture.

  • Type Grayscale / Color / Blend
  • Saturation Variable Impact
  • Scale 0 - 10px
  • Blurriness Post-process Gaussian

02. Luminance Masking

Selective application of effects based on pixel brightness values.

  • Shadow Threshold 0.0 - 1.0
  • Shadow Fade Soft Edge Control
  • Highlight Threshold 0.0 - 1.0
  • Invert Logic Boolean Toggle

03. Optics & Distortion

Simulates physical lens properties and signal errors.

  • Chromatic Aberration RGB Split / Falloff
  • Blur Types Gaussian / Box / Motion
  • Center Point Interactive 2D Coordinates
  • Edge Blur Vignette Simulation

04. Stylization

Algorithms for reducing color depth and simulating print media.

  • Dithering Bayer / Noise (Random)
  • Halftoning Dot Size / Intensity
  • Cell Shading Quantize Luma / Sobel Edges
  • Corruption JPEG / Pixelation / Bleed

System Architecture

The application maintains a persistent state object describing the render stack. Changes to input values trigger a re-render of the specific WebGL layer or the entire composite chain.

Adjustments Pipeline

Standard image correction logic applied before texture effects.

  • Brightness/Contrast -100 to 200
  • Warmth +/- 500 Shift
  • Sharpening Convolution Kernel

Color Exclusion Module:
Allows masking specific hex colors from adjustments or blurs. Uses vector distance calculation in shader space.

State Management (JSON)

The entire UI state is serializable. This permits saving workspaces and reloading exact grain parameters.

{ "noise": { "strength": 40.0, "type": 1, "blendMode": 1 }, "adjust": { "brightness": 0, "contrast": 10, "warmth": -20 }, "ca": { "amount": 15.0, "center": [0.5, 0.5] } }

Performance Constraints

High Quality Preview

By default, the viewport clamps internal textures to 2048px to maintain 60FPS on mid-range hardware. Disabling this toggle processes the image at full native resolution.

Browser Capability

Max texture size is queried via gl.getParameter(gl.MAX_TEXTURE_SIZE). Operations are shader-based; performance scales with pixel count, not layer complexity.
