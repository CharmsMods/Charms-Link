GPU Color Key & Patch Engine

A client-side image processing tool designed for chromatic keying and localized texture patching. Execution occurs entirely in the browser via WebGL fragment shaders, ensuring zero data transit.

Launch Interface

01. Chromatic Distance

Keying logic relies on Euclidean vector distance in RGB color space.

  • Algorithm GLSL Distance()
  • Masking Smoothstep Interpolation
  • Tolerance Variable Radius
  • Blending Linear Alpha Mix

02. Patch System

Supports up to 32 independent rectangular patches rendered simultaneously in a single pass.

  • Geometry AABB Collision
  • Positioning UV Coordinate Mapping
  • Interaction Drag / Resize
  • Rendering Conditional Fragment

03. Data Architecture

Application state is centralized in a JavaScript object, synchronized to the GPU via Uniform arrays.

  • Storage Float32Array
  • Transport Uniform Vector 4
  • State Serializable JSON
  • Privacy Local Only

04. Performance

Operations are parallelized on the GPU. Performance scales with image resolution, not layer complexity.

  • Context WebGL 1.0
  • Filtering Linear / Mipmap
  • Target 60 FPS Refresh
  • Buffer Preserve Drawing Buffer

Fragment Shader Logic

The core visual manipulation occurs in the fragment shader. Below is the logic responsible for determining pixel transparency based on color distance and patch overlap.

// Calculate distance from current pixel to target key color
float dist = distance(color.rgb, u_targetColor);

// Create a soft mask based on tolerance threshold
float mask = 1.0 - smoothstep(0.0, u_tolerance, dist);

// Apply patching system if enabled
if (u_patchEnabled) {
  for (int i = 0; i < 32; i++) { if (i >= u_numPatches) break;
    // Check if pixel is within patch bounding box
    vec4 rect = u_patchRects[i];
    if (pixelCoord.x >= rect.x && pixelCoord.x <= rect.x + rect.z &&
        pixelCoord.y >= rect.y && pixelCoord.y <= rect.y + rect.w) {
        // Override pixel color with patch color
        gl_FragColor = vec4(u_patchColors[i], 1.0);
        return;
    }
  }
}

// Mix original alpha with calculated transparency
float newAlpha = mix(color.a, u_targetAlpha, mask);
gl_FragColor = vec4(color.rgb * newAlpha, newAlpha);

System Constraints

Input Handling

The application accepts standard image formats (JPG, PNG). Upon upload, the source is converted to a WebGL Texture. Coordinates are normalized to 0.0 - 1.0 UV space for shader processing.

Output

The final buffer is preserved using preserveDrawingBuffer: true. This allows standard canvas.toDataURL() extraction for high-fidelity PNG downloads with retained alpha channels.
